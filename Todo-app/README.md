# Todoアプリ（JavaScript / ブラウザ版）

## 概要
ブラウザ上で動くシンプルなTodoアプリです。  
タスクの追加・完了チェック・削除ができます。  
「HTMLとJavaScriptがどうつながるのか」を、自分の手で確認するために作りました。

## 機能
- テキスト入力欄にタスクを入力して「追加」ボタンを押すと、下のリストに表示される
- Enterキーでもタスクを追加できる
- チェックボックスをONにすると、そのタスクが「完了」として見た目が変わる（取り消し線＋少し薄く表示）
- 「削除」ボタンで、1つずつタスクを削除できる

## 使っている主な技術
- HTML：入力欄・ボタン・リストの構造
- CSS：削除ボタンの余白、完了タスクの見た目（取り消し線・透明度）
- JavaScript：
  - `<script src="script.js"></script>` による HTML と JS の連携
  - `document.querySelector()` による DOM 要素の取得
  - 関数（`addTodo`）で「タスクを1件追加する処理」を1つの箱にまとめる
  - `addEventListener("click", ...)` や `addEventListener("change", ...)` でイベントを扱う
  - `classList.toggle()` を使った状態（完了/未完了）の見た目切り替え

## 自分がつまづいていたポイント
- 最初はそもそも **HTML と JavaScript がどうつながっているのか** 分かっていませんでした。
- 画面が真っ白なまま何も表示されず、「どこから確認すればいいのか」も分かりませんでした。

今回の実装では、

- `index.html` と `script.js` を同じフォルダに置く
- `index.html` の一番下に `<script src="script.js"></script>` を書く
- ブラウザで **正しい `index.html` を開く**

という「超基本」の確認を意識的に行いました。

## 講師として意識している点
自分自身が、

- 「何も表示されない」
- 「どのファイルを開けばいいか分からない」
- 「`<script>` の意味が分からない」

という状態からスタートしたので、  
同じように詰まっている初学者と同じ目線で話せることが強みだと思っています。

このTodoアプリは、単に動くものを作るだけでなく、

- 画面に何も出ないときに何を確認するか
- HTMLとJSの関係をどうイメージすると理解しやすいか
- 機能を1つずつ積み上げて「アプリの形」にしていくプロセス

を説明するための題材として位置づけています。
