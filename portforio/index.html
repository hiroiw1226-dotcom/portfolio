自己紹介とサブ講師としてのスタンス

独学で JavaScript と Python を学び、Todoアプリを題材に 「小さな機能を積み上げることでアプリは形になる」ことを体感しました。

初心者のつまずきが分かる
画面が真っ白なまま何も表示されない、
何が間違っているのか見当もつかない、
「自分には無理かも」と感じる――その感覚を実体験として理解しています。
質問を否定しない・置いてけぼりにしない
「こんなこと聞いていいのかな」という空気を壊し、 どんな質問でもまず受け止めることを大切にします。 分からなくなった地点まで一緒に戻り、階段を作り直すイメージで伴走します。
不安を小さな成功体験に変える
「とりあえず1回動かす」「1行だけ直してみる」など、 小さな成功を一緒に積み上げることで、 学習を継続できる状態を作ることを意識しています。
AI 時代の学び方
AIでコードを書くこともできますが、 初心者のつまずきに寄り添うために、Todoアプリは自分で1から手を動かして実装しました。 「なぜこう書くのか」を説明できることを大事にしています。

作品① ブラウザ版 Todo アプリ（JavaScript）

JavaScript HTML / CSS DOM操作 イベント

概要

  • タスクを入力して追加ボタン（または Enter）でリストに表示
  • チェックボックスで「完了」状態を切り替え（取り消し線＋薄く表示）
  • 削除ボタンで1件ずつタスクを削除

学べるポイント（初心者目線）

  • <script src="script.js"> で HTML と JS がつながるイメージ
  • document.querySelector() で「画面の部品を地図から探す」感覚
  • function addTodo() で「動きを箱にまとめる」関数の考え方
  • addEventListener("click", ...) や ("change", ...) による「押したら動く」仕組み
  • classList.toggle("completed") で、ロジックと見た目を分ける設計
ここでの気づき 画面に何も出なかった状態から、
「たった1つの <script> の書き忘れで止まっていた」ことに気づいた経験が、 初心者の白画面問題をどう一緒に確認していくか、という視点につながりました。

作品② ターミナル版 Todo アプリ（Python）

Python 標準入力 リスト / 辞書 エラーと仲良くなる

概要

  • ターミナル上で動く Todo アプリ
  • メニューから「追加 / 一覧 / 完了 / 削除 / 終了」を選択
  • タスクは {"text": "...", "done": True/False} の形でリストに保存

学べるポイント（初心者目線）

  • python main.py でスクリプトを実行する流れ
  • input() と while True: による対話的なアプリの構造
  • list と dict を使ったシンプルな状態管理
  • try / except で「番号ミスなどのよくあるエラー」を安全に扱う
ここでの気づき Python のインストールや、cd でフォルダを移動するところからつまずいたことで、 「環境構築そのものが初心者にとって大きなハードル」であると実感しました。 そのため、ターミナルの使い方も含めて、
一緒に一歩ずつ確認していくスタイルを大事にしたいと考えています。

サブ講師として提供できる価値

大事にしていること

  • 「何が間違っているか分からない」状態の不安を理解している
  • まず「これ、私も困りました」と共感から入る
  • 専門用語だけで押し切らず、イメージと言葉で整理する
  • 質問を否定せず、「分からないと言える空気」を作る
  • いきなり全部ではなく、小さな成功体験を積み重ねる設計をする

AI 時代のサブ講師として

  • AIを使えば難しいコードも書けますが、
    初心者の不安やつまずきはAIには体験できない部分だと考えています。
  • Todoアプリはあえて自分で1から実装し、
    「どこで止まったのか」「どうやって動くところまで辿り着いたのか」を言語化しました。
  • 今後は、AIの出したコードも含めて、
    「なぜこう動くのか？」を一緒に分解・説明できるサブ講師を目指します。

白画面から「動いた瞬間」へ – ストーリー

最初に画面が真っ白だったとき、 何を直せばいいのか見当もつかず、「全部間違っている気がして」本気で不安になりました。

しかし、ファイルの場所を1つ確認しただけで動いたとき、 その小さな成功体験が「自分でも前に進めるかもしれない」という感覚に変わりました。

この体験から、私は 「動いた瞬間」を一緒に迎えることが、学習を続けるエネルギーになると考えるようになりました。 その瞬間を作るためのサポートができるサブ講師でありたいです。
